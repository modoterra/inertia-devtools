# Contributing to Inertia Devtools

Thanks for contributing.

## Ground Rules

- Keep pull requests small and focused.
- Follow the existing project structure and style.
- Include validation steps in the PR description.

## Required Before Opening a PR

Run and pass:

```bash
npm install
npm run build
```

## Commit Requirements

### Conventional Commits (required)

Use Conventional Commit messages, for example:

- `feat: add network panel inspector`
- `fix: prevent duplicate message handling`
- `docs: improve setup instructions`

### Signed Commits (required)

All commits must be signed.

## Pull Request Expectations

- Explain why the change is needed.
- Describe how the change was validated.
- Keep one concern per PR where possible.

## Code of Conduct

By participating, you agree to follow `CODE_OF_CONDUCT.md`.
